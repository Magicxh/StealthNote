# Stealth Note 开发日志

## v2.7.1 - 2026-07-11

### 本次更新内容
基于 v2.7.0 修复三个问题：

- **修复手柄不可见**：handle_win 从 `Toplevel(root)` 改为 `Toplevel()`（无父窗口），避免 root 的 COLORKEY 透明属性影响子窗口渲染；新增 `-topmost True` 确保始终可见
- **修复四角边框遮挡文字（彻底方案）**：将四角画布/热区的创建从 `_init_corners` 移到 `_init_content` 中，**在 text_container 之前创建**，确保四角元素天然位于文本之下，不依赖 `lower()` 调用
- **修复仪表盘小红点混乱**：根因是 `_init_panel` 创建红点后从未调用 `_update_panel_button_states` 初始化状态，所有红点处于隐藏状态；现于 `_init_panel` 末尾添加初始化调用；同时移除 `_layout_all` 中过早的无效调用（此时面板尚未创建）
- **版本升级**：版本号升级为 `v2.7.1`

### 根因分析
1. 手柄：root 设为 COLORKEY 透明后，`Toplevel(root)` 创建的子窗口在 Windows 下可能受父窗口分层属性影响而不可见
2. 四角：`_init_content` → `_init_corners` 的调用顺序导致四角画布创建晚于文本容器，天然位于文本之上；`lower()` 不可靠
3. 小红点：`_init_panel` 末尾缺少 `_update_panel_button_states()` 调用，红点从未被初始化

## v2.7.0 - 2026-07-11

### 本次更新内容
**架构翻转：彻底消除黑底，同时保持文本交互顺滑。**

- **架构翻转**：从 "root 用 LWA_ALPHA + content_win 用 alpha=255" 翻转为 "root 完全透明（COLORKEY）+ content_win 用 LWA_ALPHA(bg_op)"
- **统一背景色**：content_win 内所有子元素（content_frame、text_container、stealth_container、四角画布/热区、滚动条）统一使用 `raw_bg` 实色
- **消除双重混合**：只有一层 LWA_ALPHA（content_win），无双重 alpha 混合，文本区与周围颜色完全一致
- **文本交互顺滑**：Text 控件 bg 为 `raw_bg` 实色，原生 Tkinter 鼠标/键盘交互正常工作
- **版本升级**：版本号升级为 `v2.7.0`，互斥锁名称同步更新

### 根因分析
旧架构：root 用 LWA_ALPHA(bg_op) 显示 raw_bg 半透明，content_win 用 alpha=255 全不透明，text bg = raw_bg 实色。
→ 文本区 raw_bg 全不透明 vs 周围 raw_bg 半透明 → 文本区更暗/更亮，呈现色块（黑底）

新架构：root 完全透明（COLORKEY），content_win 用 LWA_ALPHA(bg_op) 统一控制所有内容半透明，所有子元素 bg = raw_bg。
→ 只有一层 LWA_ALPHA，所有区域同一颜色同一透明度，无黑底

## v2.6.5 - 2026-07-11

### 本次更新内容
基于 v2.6.4 回滚后重新梳理，以多角色分析定位根因并逐项修复：

- **修复黑底问题**：content_win 从 `LWA_ALPHA(bg_op) + LWA_COLORKEY` 回退为 `LWA_COLORKEY only (alpha=255)`，消除 root 与 content_win 双重 alpha 混合导致的文本区变暗；文本框背景使用 `raw_bg` 原始色实色（不预乘黑），保证原生交互顺滑且无色差
- **修复小红点指示器**：dot canvas 背景从 `COLORKEY`（子控件中不透明，显示为黑块）改为按钮容器的实际 bg 色，红点直接显示在按钮上无黑块
- **修复隐写三行半行**：移除 `_refresh_stealth_view` 中 `see("insert")` 与 `yview_moveto` 的二次滚动冲突，改为 `see(top行)` + `dlineinfo` 像素级 `yview_scroll` 精确对齐；`_calc_stealth_window_height` 改用 `stealth_text.cget("font")` 获取实际渲染字体，避免 Font 对象不一致
- **修复四角框遮挡文字**：视觉画布 `lower()` 到文本容器之下，热区 frame 保留在顶层用于缩放交互
- **修复手柄圆心错位**：`_set_handle_region` 改为接收 `hs` 参数（不读 stale `winfo_width/height`），`_layout_handle` 在 `geometry()` 后加 `update_idletasks()` 确保尺寸生效
- **修复手柄黑盘+多余圆形**：handle_win/canvas 背景从 `ROOT_BG` 改为 `COLORKEY`，启用 `LWA_COLORKEY` 色键透明；未获焦时 `fill=""` 空心（中心透明），移除 `ROOT_BG` 预乘和中央白色小点
- **优化右键菜单图标**：1/2/3 行模式选择从 `add_command`（`═`/`≡` 字符渲染为黑块）改为 `add_radiobutton` + `IntVar`，自带选中指示器，`_set_stealth_lines` 同步更新选中状态
- **版本升级**：版本号升级为 `v2.6.5`，互斥锁名称同步更新

### 根因分析
1. 黑底：root 和 content_win 都对 raw_bg 做 LWA_ALPHA 混合 → 双重变暗
2. 小红点：COLORKEY 在 Tkinter 子控件中不透明 → 黑色方块
3. 三行半行：`see("insert")` 覆盖 `yview_moveto` 定位 → 二次滚动；行数 ratio 与像素 ratio 不一致
4. 四角遮挡：`lift()` 将视觉画布提到文本之上
5. 手柄圆心：`winfo_width()` 返回 stale 值 → 裁剪圆心与绘制圆心错位
6. 手柄黑盘：`ROOT_BG` 填充 + `alpha=255` 无 colorkey → 黑色圆盘
7. 菜单黑块：`═`/`≡` Unicode 字符在某些字体下渲染为方块
